# Site Builder

## Шаблоны проектов

Список шаблонов в UI строится из подпапок `modules/source/`. Служебные каталоги `theme-presets` и `placeholder-assets` не считаются шаблонами. Чтобы добавить шаблон, скопируйте структуру как у `default-template` в новую папку внутри `modules/source/`.

## Google Search Console (верификация)

В карточке проекта нажмите **«Выбрать HTML»** — откроется стандартный выбор файлов. Отметьте один или несколько `.html` с компьютера; они загружаются в `projects/<проект>/public/` с исходными именами. Остальные файлы в `public` не затрагиваются.

## Автогенерация сайта

На форме **«Создание проекта»** можно выбрать **Автогенерация** вместо ручного режима. После заполнения формы система на сервере:

1. Подбирает случайную цветовую тему и набор блоков (Main, Casino, Bet, App, Login + опциональные кастомные страницы).
2. Генерирует тексты и SEO для всех выбранных языков (один GPT-запрос на страницу/локаль).
3. Генерирует изображения (2–3 на ключевые страницы, 1–2 на Login), логотип и favicon.
4. Собирает production `dist`, создаёт ZIP и загружает на указанный SFTP/FTP сервер.

Прогресс отображается на странице проекта (polling `/api/build/project/:name/auto-status`). Пароль сервера не сохраняется; список серверов без пароля — в `data/deploy-servers.json`. Новые цветовые пресеты добавляются как `.css` в `modules/source/theme-presets/`.

### Логи автогенерации

Подробные логи пишутся **на сервере** (не в браузере):

| Где | Что |
|-----|-----|
| Терминал с `npm run dev` | строка `[server]` — все `console.log` бэкенда, в т.ч. `[text-generation]`, `[image-generation]`, `[auto-generation:имя-проекта]` |
| `projects/<имя-проекта>/auto-generation.log` | журнал только этого проекта |

В UI виден только итоговый статус шагов и текст ошибки из `project-settings.json`. Детали (модель OpenAI, Runware, какая страница/картинка) — в файлах выше. При `npm run dev` смотрите вкладку/окно терминала с меткой **server**, не client.

Опционально в `.env`: `OPENAI_MODEL_FAQ=gpt-4o-mini` — отдельная модель для FAQ, если `gpt-5` иногда возвращает пустой ответ.

## Загрузка на сервер

В карточке проекта: **Хост**, **Имя пользователя**, **Пароль**, **Порт** (22 = SFTP, 21 = FTP), **Путь на сервере**. Кнопка **«Загрузить на сервер»** собирает `dist` и **копирует файлы** в указанный каталог по SFTP/FTP. Это не «переключатель» виртуальных хостов: если nginx/apache отдаёт сайт из `/var/www/html`, а загрузка ушла в домашнюю директорию пользователя SSH (частый случай при пути `/`), по IP будет по-прежнему старый контент до тех пор, пока не укажете реальный document root или не переложите файлы. На shared-хостингах типичный путь вида `/home/user/public_html` или путь из панели. После успешной выгрузки при необходимости сделайте жёсткое обновление страницы (Ctrl+F5) или сбросьте кеш CDN. **«Архивировать проект»** — скачать ZIP исходников (без `node_modules` и `dist`). Пароль в настройках проекта не сохраняется.

Сборка `dist`: если папка уже есть и **не новее**, чем `src/`, `public/` и ключевые файлы конфига (`package.json`, `vite.config`, и т.д.), повторный `npm run build` не запускается — деплой и скачивание production-ZIP становятся быстрее. В шаблонах скрипт `build` вызывает только `vite build` (полная проверка типов при необходимости: `npm run typecheck` в каталоге проекта).

## Превью проекта

В карточке проекта: **«Установить зависимости»** (1–5 мин) или **«Запустить»** preview (установит npm, если ещё нет). Кнопка показывает секунды ожидания. Проекты на **OneDrive** часто ставят npm дольше — при зависании выполните `npm install` вручную в `projects/<имя>/`.

## Картинки без Runware

В `.env` корня: `IMAGE_PROVIDER=placeholder` — тогда страничные картинки и опционально логотип создаются как локальные WebP-заглушки (sharp), без списания Runware. По умолчанию `runware` (нужен `RUNWARE_API_KEY`).

## Стек

Node (Express API), Vite-клиент в `client/`, переменные окружения см. `env.example`.

## Запуск локально

```bash
npm run install-all   # один раз
npm start             # то же что npm run dev: API + интерфейс
```
