# Site Builder

## Шаблоны проектов

Список шаблонов в UI строится из подпапок `modules/source/`. Служебные каталоги `theme-presets` и `placeholder-assets` не считаются шаблонами. Чтобы добавить шаблон, скопируйте структуру как у `default-template` в новую папку внутри `modules/source/`.

## Google Search Console (верификация)

Все HTML-файлы верификации сложите в **одну** подпапку:

`modules/google-accounts/<любое_имя>/*.html`

В карточке проекта выберите эту папку и нажмите **«Загрузить все .html»** — файлы копируются в `projects/<проект>/public/` с исходными именами. Остальные файлы в `public` не трогаются; удаляются только ранее отслеженные верификационные HTML, которых нет в новой выгрузке.

## Превью проекта

В папке `projects/<имя>/` перед первым запуском превью в UI выполните `npm install`, иначе сервер превью не поднимет Vite.

## Картинки без Runware

В `.env` корня: `IMAGE_PROVIDER=placeholder` — тогда страничные картинки и опционально логотип создаются как локальные WebP-заглушки (sharp), без списания Runware. По умолчанию `runware` (нужен `RUNWARE_API_KEY`).

## Стек

Node (Express API), Vite-клиент в `client/`, переменные окружения см. `env.example`.
